# llms-md-spec

> Extending llms.txt for academic papers, repositories, APIs, documentation sites, and codebases — with `.md` extension and scoped filenames.

A usage guide and specification for the `llms-{scope}.md` naming convention. Built on the llms.txt standard by Jeremy Howard (llmstxt.org). Same Markdown format, scoped by filename for different audiences. CC BY 4.0 licensed.

## Install

npm packages available for each scope:

- `npm info llms-paper-md`
- `npm info llms-repo-md`
- `npm info llms-api-md`
- `npm info llms-docs-md`
- `npm info llms-code-md`

## Core Features

- [Scoped filenames]: `llms-{scope}.md` pattern separating paper, repo, api, docs, code
- [llmstxt.org compatible]: Same Markdown format as llms.txt, no spec changes
- [.md extension]: Native to developer and researcher workflows, GitHub auto-renders
- [Extensible]: Future scopes (device, env, legal) follow the same pattern
